# Plan: Dockerize Scan Tool & Remove Path Mapping

## Motivation

The scan tool currently requires two path concepts (`--real-root` and `--docker-root`) because it runs outside the Docker container while Komga runs inside it. By dockerizing the tool and running it alongside Komga with the same volume mounts, the paths are identical — no conversion needed.

## Current vs Target

| Concept | Current (dual-path) | Target (dockerized) |
|---------|---------------------|---------------------|
| Config | `real_root_path` + `docker_root_path` | `library_root_path` (single) |
| Env var | `KOMGA_REAL_ROOT` + `KOMGA_DOCKER_ROOT` | `KOMGA_LIBRARY_ROOT` |
| CLI args | `--real-root` + `--docker-root` | `--library-root` |
| Scanner URL generation | `_to_docker_path(real, real_root, docker_root)` | Use path as-is |
| Analyzer path resolution | `_docker_to_real_path(docker, real_root, docker_root)` | Use path as-is (strip `file:` only) |
| PdfAnalyzer constructor | `PdfAnalyzer(real_root, docker_root)` | `PdfAnalyzer()` or `PdfAnalyzer(library_root)` |
| Deployment | Runs on host machine | Runs inside Docker container |

## Files to Modify

### 1. `config.py` — Replace dual-path with single path

```python
@dataclass
class LibraryConfig:
    library_id: str = "0Q3CKC76902B7"
    library_root_path: str = "/data/data-books-audiobooks/Manga_Ebook/Manhwa"  # NOW SINGLE
    scan_workers: int = 0
```

Remove:
- `real_root_path` field
- `docker_root_path` field
- `KOMGA_REAL_ROOT` env var
- `KOMGA_DOCKER_ROOT` env var

Add:
- `KOMGA_LIBRARY_ROOT` env var (maps to `library_root_path`)

### 2. `scanner.py` — Remove path conversion

Current behavior:
```python
def _scan_single_series(..., real_root, docker_root, ...):
    series_url = _to_docker_path(dir_path, real_root, docker_root)
    b["url"] = _to_docker_path(b["path"], real_root, docker_root)
```

New behavior:
```python
def _scan_single_series(..., library_root, ...):
    series_url = dir_path  # absolute path, same as Komga sees it
    b["url"] = b["path"]   # absolute path, same as Komga sees it
```

Change `_scan_single_series` signature: `real_root`+`docker_root` → `library_root`.
Change `scan_library` signature: pass `library_root` instead of `real_root`+`docker_root`.
Remove `_to_docker_path()` function entirely.

The `library_root` is still needed for top-level scanning (`os.scandir(library_root)`) but NOT for path conversion anymore.

Validation: `main.py` checks `if not config.library.library_root_path` instead of checking both paths.

### 3. `analyzer.py` — Simplify to single-path

```python
class PdfAnalyzer:
    def __init__(self, library_root: str):  # or no-arg
        self._library_root = library_root.rstrip("/")

    def _resolve_path(self, docker_path: str) -> str:
        # Strip file: prefix only — no path mapping needed
        if docker_path.startswith("file:"):
            return docker_path[5:]
        return docker_path

    def analyze(self, docker_path: str, ...):
        real_path = self._resolve_path(docker_path)
        ...
```

Remove `_docker_to_real_path()` function entirely.

Update `_resolve_path()`:
- Only strips `file:` prefix (Komga occasionally stores Java URIs)
- No Docker↔real path mapping since they're the same

Update `PdfAnalyzer.__init__`:
- `library_root: str` parameter (for future use, currently only used for `file:` stripping)
- Or make it no-arg since no path conversion needed

### 5. `Dockerfile` — NEW

```dockerfile
FROM python:3.12-slim

RUN apt-get update && apt-get install -y --no-install-recommends \
    libmupdf-dev \
    && rm -rf /var/lib/apt/lists/*

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY *.py .
COPY .gitignore .

# Drop to non-root
RUN useradd -m scanner
USER scanner

ENTRYPOINT ["python", "main.py"]
```

Alternatively, use `alpine` with pre-built PyMuPDF wheel (no system deps needed since PyMuPDF bundles MuPDF):
```dockerfile
FROM python:3.12-alpine

WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY *.py .

RUN adduser -D scanner
USER scanner

ENTRYPOINT ["python", "main.py"]
```

## Path Usage Summary After Changes

| File | Old | New |
|------|-----|-----|
| `scanner.py` | `_to_docker_path(real, real_root, docker_root)` → `docker_url` | `real_path` = `docker_url` (identical) |
| `scanner.py` | `os.scandir(real_root)` | `os.scandir(library_root)` |
| `analyzer.py` | `_docker_to_real_path(docker, real_root, docker_root)` → `real_path` | Strip `file:` prefix → same path |
| `analyzer.py` | `PdfAnalyzer(real_root, docker_root)` | `PdfAnalyzer(library_root)` |
| `config.py` | `LibraryConfig.real_root_path`, `.docker_root_path` | `LibraryConfig.library_root_path` |
| `main.py` | `--real-root`, `--docker-root` | `--library-root` |
| `main.py` | Validate both | Validate single |

## Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| Production DB has `file:` prefixed URLs | Keep `file:` stripping in `_resolve_path()` |
| User needs non-dockerized mode | Keep backwards compat — if `--library-root` not set, use old `--real-root`/`--docker-root` (transition strategy) |
| PyMuPDF in Alpine | PyMuPDF wheel bundles MuPDF C library — no system deps needed. Works on Alpine. |
| Read-only volume | Tool only reads files (scan + analyze), writes only to DB. No write to library volume needed. |
