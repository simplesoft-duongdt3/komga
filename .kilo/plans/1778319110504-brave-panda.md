# Plan: Add Book Analysis (MEDIA + Page Dimensions + File Hash) to Python Scan Tool

## Research Summary

Komga's analyze task (`BookAnalyzer.kt:60-96`, invoked via `BookLifecycle.analyzeAndPersist()`) performs these steps for PDF books:

| Step | What it does | DB Tables Updated |
|------|-------------|-------------------|
| 1. Detect media type | Uses Apache Tika → `application/pdf` | MEDIA.MEDIA_TYPE |
| 2. Extract pages | PDFBox reads all pages, extracts cropBox dimensions | MEDIA_PAGE (per-page rows) |
| 3. Count pages | `pdf.numberOfPages` | MEDIA.PAGE_COUNT |
| 4. Set status | If no errors → `READY` | MEDIA.STATUS |
| 5. File hash | SHA-256 of entire file | BOOK.FILE_HASH |

Thumbnail generation (step 5 in Komga's original flow) is **skipped** — our scan tool already inserts SIDECAR JPG thumbnails from the filesystem (`poster.jpg`, `chapter-*.jpg`). Those are sufficient for the frontend.

For PDFs, `BookAnalyzer.analyzePdf()` (`BookAnalyzer.kt:232-238`) simply calls:
```kotlin
val pages = pdfExtractor.getPages(book.path, analyzeDimensions).map { BookPage(it.name, "", it.dimension) }
return Media(status = Media.Status.READY, pages = pages)
```

## Python Libraries

| Library | Purpose |
|---------|---------|
| `PyMuPDF` (fitz) | Page count, cropBox dimensions — **single dep**, no system libs |
| `hashlib` | SHA-256 file hashing (stdlib) |

## Implementation Plan

### 1. New file: `analyzer.py`

```
tool-scan-library/analyzer.py
```

**Class:**
- `PdfAnalyzer` — wraps PyMuPDF operations

**Methods:**
- `get_page_count(path) → int` — opens PDF, returns `doc.page_count`
- `get_all_pages(path) → list[dict]` — returns `[{number: 1, file_name: "1", width: W, height: H}, ...]` for all pages
- `compute_file_hash(path) → str` — SHA-256 hex digest of entire file, read in 64KB chunks

### 2. New mode: `--analyze` flag in `main.py`

```
python main.py --analyze
```

**Flow:**
1. Connect to DB (write-capable)
2. Query books with UNKNOWN media:
   ```sql
   SELECT b.*, m."STATUS" as media_status
   FROM "BOOK" b JOIN "MEDIA" m ON m."BOOK_ID" = b."ID"
   WHERE b."LIBRARY_ID" = %s AND m."STATUS" = 'UNKNOWN' AND b."DELETED_DATE" IS NULL
   ```
3. For each book (ThreadPoolExecutor, configurable workers):
   a. Open PDF with PyMuPDF → page count
   b. Extract dimensions for each page → prepare MEDIA_PAGE rows
   c. Compute SHA-256 hash → prepare BOOK.FILE_HASH update
   d. Prepare MEDIA update: STATUS='READY', MEDIA_TYPE='application/pdf', PAGE_COUNT=N
4. Batch execute all SQL operations:
   - INSERT INTO MEDIA_PAGE (with ON CONFLICT upsert)
   - UPDATE MEDIA (STATUS, PAGE_COUNT, MEDIA_TYPE, LAST_MODIFIED_DATE)
   - UPDATE BOOK (FILE_HASH)

### 3. DB operations in `db.py`

Add methods:
- `fetch_unanalyzed_books(library_id) → list[dict]` — books with MEDIA.STATUS='UNKNOWN'
- `update_media_analyzed(updates: list[dict])` — batch UPDATE MEDIA: STATUS='READY', PAGE_COUNT, MEDIA_TYPE='application/pdf'
- `insert_media_pages_batch(pages: list[dict])` — batch INSERT/upsert MEDIA_PAGE rows per page
- `update_book_hashes(updates: list[dict])` — batch UPDATE BOOK.FILE_HASH

### 4. CLI integration in `main.py`

```bash
# Analyze all UNKNOWN books for a library (standalone)
python main.py --analyze

# Full pipeline: scan + analyze in sequence
python main.py --analyze --real-root ... --docker-root ...

# Analyze only N books
python main.py --analyze --analyze-limit 100

# Analyze with specific worker count
python main.py --analyze --workers 8

# Skip hashing for speed
python main.py --analyze --no-hash

# Skip dimensions (page count only) for speed
python main.py --analyze --no-dimensions
```

New options:
- `--analyze` — run analysis phase (updates MEDIA + MEDIA_PAGE + BOOK.FILE_HASH)
- `--analyze-limit N` — process at most N books (default: all)
- `--no-hash` — skip SHA-256 file hashing (just set MEDIA status)
- `--no-dimensions` — skip page-by-page dimensions (just set PAGE_COUNT)
- Analysis uses ThreadPoolExecutor for parallel PDF processing

### 5. Performance estimates

| Operation | Time per book | For 300k books (16 threads) |
|-----------|--------------|---------------------------|
| Page count only | ~5ms | ~1.6 min |
| Page count + dimensions | ~10ms | ~3 min |
| + SHA-256 hash | ~100ms (for 30MB) | ~30 min |
| Full analysis | ~110ms | ~35 min |

Hashing is the main bottleneck. Use `--no-hash` when speed is critical.
